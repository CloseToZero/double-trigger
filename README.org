* Double Trigger

A small Emacs package for running a function after two characters are
entered quickly.

After the first key, Double Trigger waits up to =double-trigger-delay= for
the next input event.  A matching second key replaces the current command
with the trigger function.  On a timeout, the first key's normal command
runs.  A nonmatching event is returned to the command loop, where it runs
after the first key's command.

The synchronous lookahead keeps the command loop from observing a partial
key pair.  It does not insert or delete speculative text, rewrite
=buffer-undo-list=, or defer the first key through a timer.  A lone first
key can therefore appear up to =double-trigger-delay= seconds later.

** Configuration

#+begin_src emacs-lisp
(require 'double-trigger)

(setq double-trigger-keys "ii"
      double-trigger-delay 0.2
      double-trigger-fn #'keyboard-quit)

(double-trigger-mode 1)
#+end_src

Use =double-trigger-inhibit= for a temporary dynamic override, or add
zero-argument predicates to =double-trigger-inhibit-fns= to restrict the
states and modes where detection is active.

Detection starts only when the first key resolves to =self-insert-command=.
This leaves command states, read-only keymaps, and modes where the key has a
different meaning untouched.

Both =double-trigger-insert-fn= and =double-trigger-delete-fn= are retained
for compatibility but obsolete.  Detection no longer inserts or removes a
candidate key speculatively.
